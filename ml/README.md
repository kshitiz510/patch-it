# Patch-It ML Service

Pothole detection using YOLO (Ultralytics) + Django REST API.

## Quick Start

```bash
cd ml
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS/Linux

pip install -r requirements.txt
```

### Install PyTorch (GPU)

Visit https://pytorch.org/get-started/locally/ and pick your CUDA version. Example:

```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

### Run Django Server

```bash
python manage.py migrate
python manage.py runserver 8000
```

Test: `GET http://localhost:8000/api/health/`

## Getting Training Data

You need pothole images with YOLO-format bounding box labels. Here are **free datasets**:

### 1. Roboflow (Recommended - easiest)

- **Pothole Detection Dataset (8,000+ images)**
  https://universe.roboflow.com/atharva-malhotra-gy8jk/pothole-detection-yolov5
  1. Go to the link above
  2. Click **Download Dataset** → Format: **YOLOv8**
  3. Extract into `ml/data/` so the structure is:
     ```
     ml/data/
       train/images/  train/labels/
       valid/images/  valid/labels/
       data.yaml
     ```
  4. Rename `valid/` → `val/` if needed.

### 2. Kaggle Datasets

- **Pothole Detection Dataset (600+ images)**
  https://www.kaggle.com/datasets/sachinpatel21/pothole-image-dataset

- **Road Pothole Images (2,000+)**
  https://www.kaggle.com/datasets/atulyakumar98/pothole-v8-yolo

- **Indian Roads Pothole Dataset**
  https://www.kaggle.com/datasets/sovitrath/road-pothole-images-for-pothole-detection

> For Kaggle datasets, you may need to convert labels to YOLO format. Use Roboflow if you want ready-to-use YOLO labels.

### 3. Mendeley Data

- **RDD2022 - Road Damage Dataset**
  https://data.mendeley.com/datasets/5ty2wb6gvg/3

  This is a large-scale multi-country road damage dataset (26,000+ images). Includes potholes + cracks.

## Expected Data Structure

After downloading, your `ml/data/` should look like:

```
ml/data/
├── data.yaml           ← auto-generated by train.py (or from Roboflow)
├── train/
│   ├── images/         ← .jpg / .png training images
│   └── labels/         ← .txt YOLO annotation files
├── val/
│   ├── images/
│   └── labels/
└── test/               ← (optional)
    ├── images/
    └── labels/
```

Each `.txt` label file has one line per bounding box:

```
0 0.45 0.32 0.12 0.08
```

Format: `class_id center_x center_y width height` (all normalized 0-1).

If `data.yaml` is missing, `train.py` auto-generates one.

## Training

```bash
cd ml
python scripts/train.py
```

This produces `ml/models/pothole_yolo/weights/best.pt` which the Django API uses.

Training with default settings (50 epochs, batch 16, 640px) takes:

- ~15 min on RTX 3060+
- ~45 min on GTX 1650
- ~3 hours on CPU (not recommended)

## Inference

```bash
# Single image
python scripts/predict.py --image path/to/image.jpg

# Video file
python scripts/video_inference.py --input path/to/road_video.mp4 --output data/output.mp4
```
